使用 **PowerShell 脚本** 配合 **Windows 任务计划程序** 监控网络

## 方案概述

1. **PowerShell 脚本**：负责检测网络连接状态，并在网络断开时重启无线网卡
2. **任务计划程序**：设置脚本开机自启动，并配置其一直后台运行

---

## 📜 完整 PowerShell 脚本

将以下代码保存为 `NetworkMonitor.ps1`（注意：保存时编码格式选择 **UTF-8 with BOM**，否则中文注释可能导致运行异常）：

```powershell
# ============================================
# 网络监控脚本 - 整点及半点检测网络并自动修复
# 运行时间：早上7点到晚上10点
# 检测频率：每整点及30分钟（即每小时的第0和30分钟）
# ============================================

# 配置参数
$NetworkAdapterName = "WLAN"  # 请修改为你的无线网卡名称
$StartHour = 7                # 开始时间（早上7点）
$EndHour = 22                 # 结束时间（晚上10点）
$LogFile = "$env:USERPROFILE\Desktop\NetworkMonitor.log"  # 日志文件路径

# 写入日志函数
function Write-Log {
    param([string]$Message)
    $TimeStamp = Get-Date -Format "yyyy-MM-dd HH:mm:ss"
    "$TimeStamp - $Message" | Out-File -FilePath $LogFile -Append -Encoding UTF8
}

Write-Log "========================================"
Write-Log "网络监控服务已启动"
Write-Log "网卡名称: $NetworkAdapterName"
Write-Log "工作时间: $StartHour:00 - $EndHour:00"
Write-Log "========================================"

# 主循环 - 持续运行
while ($true) {
    $CurrentTime = Get-Date
    $CurrentHour = $CurrentTime.Hour
    $CurrentMinute = $CurrentTime.Minute
    
    # 检查是否在工作时间范围内（7:00 - 22:00）
    $IsWorkingHour = ($CurrentHour -ge $StartHour -and $CurrentHour -lt $EndHour)
    
    # 检查是否为整点或半点（分钟为0或30）
    $IsCheckTime = ($CurrentMinute -eq 0 -or $CurrentMinute -eq 30)
    
    if ($IsWorkingHour -and $IsCheckTime) {
        Write-Log "开始检测网络连接... (时间: $($CurrentTime.ToString('HH:mm')))"
        
        # 检测网络连接状态（尝试 ping 百度、谷歌和微软）
        $TestResult = $false
        $TestTargets = @("www.baidu.com", "www.google.com", "www.microsoft.com")
        
        foreach ($Target in $TestTargets) {
            try {
                $PingResult = Test-Connection -ComputerName $Target -Count 1 -Quiet -ErrorAction Stop
                if ($PingResult) {
                    $TestResult = $true
                    Write-Log "网络连接正常 (成功 ping 通 $Target)"
                    break
                }
            } catch {
                # 忽略 ping 失败的异常
            }
        }
        
        # 如果网络连接失败，重启无线网卡
        if (-not $TestResult) {
            Write-Log "警告: 检测到网络连接失败！正在尝试重启无线网卡..."
            
            try {
                # 获取无线网卡并重启
                $Adapter = Get-NetAdapter -Name $NetworkAdapterName -ErrorAction Stop
                Write-Log "找到网卡: $($Adapter.Name) (状态: $($Adapter.Status))"
                
                # 禁用网卡
                Disable-NetAdapter -Name $NetworkAdapterName -Confirm:$false -ErrorAction Stop
                Write-Log "已禁用无线网卡"
                Start-Sleep -Seconds 5  # 等待5秒
                
                # 启用网卡
                Enable-NetAdapter -Name $NetworkAdapterName -Confirm:$false -ErrorAction Stop
                Write-Log "已启用无线网卡，等待网络恢复..."
                Start-Sleep -Seconds 10  # 等待10秒让网卡重新连接
                
                # 验证重启后网络是否恢复
                $RecoveryTest = Test-Connection -ComputerName "www.baidu.com" -Count 2 -Quiet
                if ($RecoveryTest) {
                    Write-Log "网卡重启成功，网络已恢复！"
                } else {
                    Write-Log "网卡已重启，但网络仍未恢复，请检查网络环境或路由器状态。"
                }
            } catch {
                Write-Log "错误: 网卡重启失败 - $($_.Exception.Message)"
                Write-Log "请检查网卡名称是否正确，或以管理员权限运行脚本。"
            }
        } else {
            Write-Log "网络连接正常，无需操作。"
        }
        
        Write-Log "等待下一个检测点..."
        
        # 等待到下一个整点或半点（避免重复检测）
        $NextMinute = if ($CurrentMinute -eq 0) { 30 } else { 60 }
        $SleepSeconds = ($NextMinute - $CurrentMinute) * 60 - (Get-Date).Second
        if ($SleepSeconds -gt 0) {
            Start-Sleep -Seconds $SleepSeconds
        }
    } else {
        # 非检测时间，等待30秒后重新检查时间
        Start-Sleep -Seconds 30
    }
}
```

## ⚙️ 配置步骤

### 第一步：修改脚本配置

在运行脚本前，你需要根据实际情况修改以下几个关键配置：

| 配置项 | 说明 | 示例值 |
|--------|------|--------|
| `$NetworkAdapterName` | 无线网卡名称（必改） | `"WLAN"`、`"Wi-Fi"`、`"Intel(R) Wi-Fi 6"` |
| `$StartHour` | 开始监控时间（24小时制） | `7`（早上7点） |
| `$EndHour` | 结束监控时间（24小时制） | `22`（晚上10点） |
| `$LogFile` | 日志保存路径 | 默认保存在桌面 |

**如何查看无线网卡名称？**

在 PowerShell 中运行以下命令：
```powershell
Get-NetAdapter | Where-Object {$_.InterfaceDescription -like "*WiFi*" -or $_.Name -like "*WLAN*"} | Format-List Name, InterfaceDescription
```

### 第二步：设置脚本执行策略

以**管理员身份**打开 PowerShell，运行以下命令允许脚本执行：

```powershell
Set-ExecutionPolicy RemoteSigned -Force
```

### 第三步：配置开机自启动

使用**任务计划程序**创建开机自启任务：

**方法一：通过 PowerShell 一键创建（推荐）**

以**管理员身份**打开 PowerShell，运行以下命令（注意修改脚本的实际存放路径）：

```powershell
# 请将下面的路径修改为你保存 NetworkMonitor.ps1 的实际路径
$ScriptPath = "C:\YourPath\NetworkMonitor.ps1"

$Action = New-ScheduledTaskAction -Execute "powershell.exe" -Argument "-WindowStyle Hidden -NoProfile -ExecutionPolicy Bypass -File `"$ScriptPath`""
$Trigger = New-ScheduledTaskTrigger -AtStartup -RandomDelay (New-TimeSpan -Minutes 1)
$Settings = New-ScheduledTaskSettingsSet -AllowStartIfOnBatteries -DontStopIfGoingOnBatteries -StartWhenAvailable -RunOnlyIfNetworkAvailable -MultipleInstances IgnoreNew
$Principal = New-ScheduledTaskPrincipal -UserId "$env:USERNAME" -LogonType Interactive -RunLevel Highest

Register-ScheduledTask -TaskName "NetworkMonitorService" -Action $Action -Trigger $Trigger -Settings $Settings -Principal $Principal -Force

Write-Host "任务计划已创建，脚本将在下次开机时自动运行！" -ForegroundColor Green
```

**方法二：通过任务计划程序图形界面创建**

1. 按 `Win + R`，输入 `taskschd.msc` 回车
2. 点击右侧的 **“创建任务”**
3. **常规** 选项卡：
   - 名称：`NetworkMonitorService`
   - 勾选 **“使用最高权限运行”**
4. **触发器** 选项卡：
   - 新建 → 开始任务：`启动时` → 确定
5. **操作** 选项卡：
   - 新建 → 操作：`启动程序`
   - 程序：`powershell.exe`
   - 参数：`-WindowStyle Hidden -NoProfile -ExecutionPolicy Bypass -File "C:\YourPath\NetworkMonitor.ps1"`
6. **条件** 选项卡：
   - 取消勾选 **“只有在计算机使用交流电源时才启动此任务”**
   - 勾选 **“只有在以下网络连接可用时才启动”** → 选择 `任何连接`
7. **设置** 选项卡：
   - 勾选 **“允许任务按需运行”**
   - 勾选 **“如果任务失败，按以下频率重新启动”** → 间隔：`5分钟` → 最多尝试：`3次`
8. 点击 **确定** 完成创建

## 🧪 测试方法

### 手动测试脚本是否正常工作

1. 临时关闭 WiFi 或拔掉网线
2. 手动运行脚本，观察日志文件（在桌面生成 `NetworkMonitor.log`）
3. 日志应显示检测到网络失败，并尝试重启网卡

### 查看运行日志

脚本运行日志默认保存在桌面 `NetworkMonitor.log` 文件中，可以用记事本打开查看。

## ⚠️ 注意事项

1. **管理员权限**：脚本中的网卡重启操作需要管理员权限，请确保任务计划使用最高权限运行。

2. **网卡名称准确性**：脚本中的 `$NetworkAdapterName` 必须与系统中的无线网卡名称完全一致（区分大小写）。

3. **省电模式干扰**：如果笔记本开启了省电模式，可能会阻止网卡重启。建议在电源选项中关闭“允许计算机关闭此设备以节约电源”。

   - 打开 **设备管理器** → **网络适配器** → 找到无线网卡 → **右键属性** → **电源管理** → 取消勾选 **“允许计算机关闭此设备以节约电源”**

4. **后台运行**：脚本运行时会在后台持续运行，在任务管理器中可以看到 `powershell.exe` 进程。

5. **停止脚本**：如果需要停止脚本，可以在任务管理器中结束 `powershell.exe` 进程，或在任务计划程序中禁用 `NetworkMonitorService` 任务。

## 🔧 故障排查

| 问题 | 可能原因 | 解决方法 |
|------|----------|----------|
| 脚本无法运行 | 执行策略未设置 | 运行 `Set-ExecutionPolicy RemoteSigned` |
| 网卡重启失败 | 网卡名称不正确 | 用 `Get-NetAdapter` 确认正确名称 |
| 任务计划不执行 | 权限不足 | 确保勾选“使用最高权限运行” |
| 日志没有生成 | 脚本未启动 | 检查任务计划程序中的任务状态 |
